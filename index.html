Sobre o Projeto

Este projeto foi criado para demonstrar a gestão de um time fictício no GitHub, com organização via Issues, Pull Requests e Project Board. O objetivo deste projeto é demonstrar, na prática, como funciona a colaboração em equipe com GitHub: criação de Issues, organização em um Project (Kanban), uso de branches, Pull Requests e revisões. Além disso, o projeto busca simular o fluxo real de trabalho de uma equipe de desenvolvimento, desde a definição de tarefas até a entrega final. Cada problema proposto foi tratado como uma demanda real, passando pelas etapas de planejamento, execução, revisão e integração. O repositório também foi estruturado de forma a destacar boas práticas de versionamento e comunicação entre membros da equipe. Dessa forma, foi possível exercitar tanto aspectos técnicos, como o uso de HTML e CSS, quanto habilidades de organização e gestão de tarefas. Assim, este projeto serve não apenas como um exemplo prático de código, mas também como um estudo guiado sobre metodologias ágeis e uso eficiente do GitHub para trabalho colaborativo.
